# Mnemo v3 — 功能拓展白皮书：减负 + 留存


> **版本**: v3.0-draft
> **更新日期**: 2026-04-21
> **定位**: 从"记忆工具"到"学习中枢"——让用户不再需要同时打开 5 个 APP

---

## 0. 核心洞察

### 0.1 用户痛点：APP 疲劳

一个考研学生的手机上通常有：

```
📱 当前状态（5+ 个 APP）：
  ├── Anki / Quizlet    → 间隔重复记忆
  ├── 背书匠            → AI 监督背诵
  ├── 不背单词 / 墨墨   → 单词记忆
  ├── 滴答清单          → 任务管理
  ├── 番茄钟 APP        → 专注计时
  ├── 网盘 / 微信文件   → 学习材料管理
  └── 各课程 APP        → 听课笔记
```

**问题**：用户在多个 APP 之间切换，数据割裂，记忆负担重。
Mnemo 的目标不是做"又一个记忆 APP"，而是**替代上述大部分工具**，成为唯一的学习中枢。

### 0.2 减负原则

```
减负 = 减少认知负担 + 减少操作步骤 + 减少切换成本

具体策略：
  1. 智能推荐：用户打开 APP 就知道今天该做什么，不需要自己规划
  2. 融合训练：一次训练覆盖多种题型，不需要分别进入不同模式
  3. 被动学习：利用碎片时间（排队、等车）自动推送轻量复习
  4. 无感记录：学习行为自动记录，不需要手动打卡
  5. 一站式：导入材料 → 自动生成卡片 → 自动安排复习 → 自动出题
```

### 0.3 留存策略：替代竞品的核心功能

| 竞品 | 用户为什么用它 | Mnemo 如何替代 | 优先级 |
|------|---------------|---------------|--------|
| **Anki** | 间隔重复、丰富题型 | CASR 算法 + 多题型 + Anki 导入 | P0 |
| **Quizlet** | AI 出题、学习游戏 | AI 自动出题 + 连击系统 + 游戏化 | P0 |
| **背书匠** | AI 监督背诵 | ASR 卡顿检测 + TTS 渐进提示 | P1 |
| **不背单词** | 语境记忆、学习曲线 | 语境例句 + 可视化遗忘曲线 | P1 |
| **滴答清单** | 任务管理、日历 | 已有任务 + 番茄钟 + 习惯打卡 | P1 |
| **番茄钟 APP** | 专注计时 | 内置番茄钟 + 学习时长统计 | P1 |
| **Notion** | 知识库、笔记 | 学习笔记 + 卡片联动 | P2 |

## 1. Phase 4：题型拓展（最高优先级）

> 白皮书 v1 规划了 8 种训练模式，v2 实现了 5 种（standard/write_en_to_zh/write_zh_to_en/cloze/paragraph）。
> 还缺：选择题、判断题、排序题、配对题。这些是 Anki/Quizlet 用户最常用的题型。

### Sprint 16：选择题 + 判断题（5 天）

#### 16.1 选择题（Multiple Choice）

```
触发方式：
  ├── 卡片集详情 → [选择题训练]
  ├── AI 自动出题（基于材料内容生成）
  └── 训练模式切换器

题目生成策略：
  ├── 自动生成：基于卡片内容，LLM 生成 4 个选项
  │   ├── 正确答案：卡片的 target_text
  │   ├── 干扰项 1：同领域易混淆项（从同卡片集随机抽取）
  │   ├── 干扰项 2：LLM 生成相似但错误的翻译/表述
  │   └── 干扰项 3：LLM 生成部分正确但有关键错误的选项
  │
  └── 手动创建：用户在卡片编辑时添加选项（可选）

UI 设计：
  ┌─────────────────────────────────────────┐
  │  Q3/20              ✗2  ✓1  ~0   ⏱ 00:45│
  ├─────────────────────────────────────────┤
  │                                         │
  │  "普惠金融"的正确英文翻译是？             │
  │                                         │
  │  ┌─────────────────────────────────┐   │
  │  │  ○ A. inclusive education       │   │
  │  │  ○ B. inclusive finance         │   │
  │  │  ○ C. universal finance         │   │
  │  │  ○ D. public finance            │   │
  │  └─────────────────────────────────┘   │
  │                                         │
  │  [跳过]                    [确认选择]    │
  └─────────────────────────────────────────┘

答题后反馈：
  ├── 正确：选项变绿 + ✓ 图标弹入 + "正确！+10分"
  ├── 错误：选中项变红 + 正确项变绿 + 显示解析
  └── 超时：自动跳过，标记为"未掌握"
```

#### 16.2 判断题（True/False）

```
题目生成策略：
  ├── 自动生成：基于卡片内容，LLM 生成正确/错误表述
  │   ├── 正确题：直接使用卡片内容
  │   └── 错误题：替换关键词（同义词替换、词序调换、概念混淆）
  │
  └── 比例：50% 正确 + 50% 错误

UI 设计：
  ┌─────────────────────────────────────────┐
  │  Q7/20              ✗2  ✓4  ~1   ⏱ 01:20│
  ├─────────────────────────────────────────┤
  │                                         │
  │  判断以下翻译是否正确：                   │
  │                                         │
  │  "外汇储备" → foreign exchange reserves  │
  │                                         │
  │  ┌──────────┐  ┌──────────┐             │
  │  │  ✓ 正确  │  │  ✗ 错误  │             │
  │  └──────────┘  └──────────┘             │
  │                                         │
  └─────────────────────────────────────────┘

答题后反馈（错误时）：
  ┌─────────────────────────────────────────┐
  │  ✗ 回答错误                              │
  │                                         │
  │  正确答案：✓ 正确                        │
  │  "外汇储备"的英文确实是 foreign exchange  │
  │  reserves，这是标准译法。                 │
  │                                         │
  │  [下一题]                                │
  └─────────────────────────────────────────┘
```

#### 16.3 数据模型

```sql
-- 选择题选项表
CREATE TABLE mc_options (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    card_id UUID NOT NULL REFERENCES memory_cards(id) ON DELETE CASCADE,
    option_text TEXT NOT NULL,
    is_correct BOOLEAN NOT NULL DEFAULT FALSE,
    sort_order INT NOT NULL DEFAULT 0,
    created_at TIMESTAMP DEFAULT NOW()
);

-- 判断题表
CREATE TABLE tf_questions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    card_id UUID NOT NULL REFERENCES memory_cards(id) ON DELETE CASCADE,
    statement TEXT NOT NULL,          -- 题目表述
    is_correct BOOLEAN NOT NULL,      -- 正确答案
    explanation TEXT,                 -- 解析
    created_at TIMESTAMP DEFAULT NOW()
);

-- 练习记录表（统一所有题型）
CREATE TABLE practice_records (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id),
    card_id UUID REFERENCES memory_cards(id),
    question_type VARCHAR(20) NOT NULL,  -- 'mc' | 'tf' | 'cloze' | 'translate' | 'standard'
    result VARCHAR(20) NOT NULL,        -- 'correct' | 'wrong' | 'skipped'
    time_spent_ms INT,                  -- 答题用时
    session_id UUID,                    -- 所属训练会话
    created_at TIMESTAMP DEFAULT NOW()
);
```

#### 16.4 API

```
POST /api/v1/memory/train/mc/start
  body: { card_set_id?, count?: 20, difficulty?: 'mixed' | 'easy' | 'hard' }
  response: { session_id, questions: [{ id, card_id, prompt, options, correct_index }] }

POST /api/v1/memory/train/mc/answer
  body: { session_id, question_id, selected_index, time_spent_ms }
  response: { correct, correct_index, explanation, score_delta }

POST /api/v1/memory/train/tf/start
  body: { card_set_id?, count?: 20 }
  response: { session_id, questions: [{ id, card_id, statement, correct_answer }] }

POST /api/v1/memory/train/tf/answer
  body: { session_id, question_id, user_answer, time_spent_ms }
  response: { correct, explanation, score_delta }
```

### Sprint 18：AI 自动出题引擎（7 天）

> 这是替代 Quizlet 的核心功能。导入材料后，AI 围绕内容自动生成多种题型。

#### 18.1 出题引擎架构

```
用户选择卡片集 → 点击 [AI 出题]
  │
  ├─ 后端将材料文本 + 用户选择的题型发送给 LLM
  ├─ LLM 返回结构化 JSON（题目 + 选项 + 正确答案 + 解析）
  ├─ 题目存入数据库（可复用，避免重复生成）
  ├─ 前端渲染题目列表
  │
  ├─ 用户逐题作答
  ├─ 选择题/判断题：即时判定
  ├─ 填空题：提交后比对 + 模糊匹配
  ├─ 排序题：提交后比对顺序
  └─ 简答题：提交后 LLM 评分 + 反馈
  │
  └─ 做题结束 → 统计面板
      ├─ 各题型正确率
      ├─ 薄弱知识点（LLM 分析）
      ├─ 错题自动加入错题本
      └─ [重新出题] / [只练错题]
```

#### 18.2 LLM Prompt 设计

```
系统 Prompt：
  你是一个专业的学习出题助手。根据提供的材料内容，生成高质量的学习题目。

  约束：
  1. 题目必须严格基于材料内容，不编造事实
  2. 选择题干扰项要有迷惑性（同类易混淆项）
  3. 难度梯度：简单(30%) → 中等(50%) → 困难(20%)
  4. 每题附带简短解析
  5. 输出严格 JSON 格式

  输出格式：
  {
    "questions": [
      {
        "type": "mc",           // mc | tf | cloze | ordering | short_answer
        "difficulty": "easy",   // easy | medium | hard
        "prompt": "...",
        "options": ["A", "B", "C", "D"],  // 仅 mc
        "correct_answer": "B",            // mc: index, tf: true/false, cloze: word
        "explanation": "..."
      }
    ]
  }
```

#### 18.3 题库管理

```
题库功能：
  ├── 自动生成：AI 基于材料生成题目
  ├── 手动创建：用户自己出题
  ├── 题目编辑：修改 AI 生成的题目
  ├── 题目收藏：标记好题/难题
  ├── 难度标签：自动/手动标注难度
  ├── 质量评分：基于用户答题数据评估题目质量
  └── 去重：避免生成重复题目

题库统计：
  ├── 各题型题目数量
  ├── 各难度分布
  ├── 平均正确率
  └── 薄弱知识点 Top 10
```

### Sprint 20：被动学习 + 碎片时间利用（5 天）

#### 20.1 通知卡片（Notification Cards）

```
概念：将复习内容直接推送到通知栏，用户不需要打开 APP 就能复习。

实现方式：
  ├── Web Push / PWA 通知
  ├── 通知内容 = 一张卡片的问题面
  ├── 用户点击通知 → 展开显示答案
  └── 标记"记得/忘了" → 自动更新 CASR 调度

通知卡片示例：
  ┌─────────────────────────────────┐
```

## 3. Phase 6：深度功能

### Sprint 21：AI 监督背诵（7 天）

> 替代"背书匠"的核心功能。用户对着手机背诵，AI 实时监听并给提示。

#### 21.1 背诵流程

```
选择背诵材料 → 开始背诵
  │
  ├─ 显示材料标题/首句（提示）
  ├─ 用户开始背诵（语音输入）
  │   ├─ ASR 实时转写用户语音
  │   ├─ 与原文比对
  │   │   ├─ 完全正确 → 继续监听
  │   │   ├─ 停顿 > 3s → 渐进提示
  │   │   ├─ 错误 → 记录错误位置
  │   │   └─ 卡壳 > 5s → TTS 播报下一句
  │   └─ 背诵结束 → 评分 + 错误报告
  │
  └─ 背诵报告：
      ├─ 完整度：背诵了多少 / 总共多少
      ├─ 准确度：正确率
      ├─ 流畅度：停顿次数、平均停顿时长
      ├─ 错误位置标注（原文高亮错误处）
      └─ 改进建议
```

#### 21.2 渐进提示策略

```
提示等级（从轻到重）：
  Level 0：无提示，用户自由背诵
  Level 1：显示下一句的首字/首词
  Level 2：显示下一句的前半部分
  Level 3：TTS 播报下一句
  Level 4：显示完整下一句

触发条件：
  ├── 停顿 > 3s → Level 1
  ├── 停顿 > 5s → Level 2
  ├── 停顿 > 8s → Level 3
  ├── 用户主动请求提示 → 下一级
  └── 用户连续错误 → Level 4
```

#### 21.3 技术方案

```
ASR（语音识别）：
  ├── Web：Web Speech API / Whisper API
  ├── Tauri：系统原生 ASR（macOS SFSpeechRecognizer / Windows Speech Recognition）
  └── 流式识别：实时转写，低延迟

TTS（语音合成）：
  ├── Web：Web Speech Synthesis API
  ├── Tauri：系统原生 TTS
  └── 可选：DashScope TTS（更自然的语音）

比对算法：
  ├── 文本相似度：Levenshtein distance / difflib
  ├── 关键词匹配：必须包含的核心词汇
  ├── 语义相似度（可选）：embedding cosine similarity
  └── 容错：忽略标点、空格、大小写差异
```
